Pętla for:


Konstrukcja:
for (inicjalizacja; warunek; zmiana) {
// kod do wykonania
}
Opis
Pętla for służy do wielokrotnego wykonywania bloku kodu przy znanej liczbie powtórzeń. Składa się z trzech części:
- Inicjalizacja (np. int i = 0) – wykonywana raz na początku
- Warunek (np. i < 10) – sprawdzany przed każdą iteracją
- Zmiana (np. i++) – wykonywana po każdej iteracji
Schemat blokowy:
Start → Inicjalizacja → Sprawdź warunek → Jeśli prawda → Wykonaj kod → Zmień wartość → Powrót do warunku Jeśli fałsz → Koniec

Pętla while z kontrolowanym wejściem


Konstrukcja:
while (warunek) {
// kod do wykonania
}
Opis:
Pętla while wykonuje blok kodu tak długo, jak warunek logiczny jest spełniony.
Warunek jest sprawdzany przed każdą iteracją.
Pętla może nie wykonać się ani razu, jeśli warunek od razu jest fałszywy.
Schemat blokowy:
Start → Sprawdź warunek → Jeśli prawda → Wykonaj kod → Powrót do warunku
Jeśli fałsz → Koniec

Pętla do while z kontrolowanm wyjściem


Konstrukcja:
do {
// kod do wykonania
} while (warunek);
Opis:
Pętla do while zawsze wykona się co najmniej raz,
ponieważ warunek jest sprawdzany po wykonaniu kodu. Używana wtedy,
gdy przynajmniej jedno wykonanie jest konieczne niezależnie od warunku.
Schemat blokowy:
Start → Wykonaj kod → Sprawdź warunek → Jeśli prawda → Powrót do kodu
Jeśli fałsz → Koniec
